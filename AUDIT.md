## Repo location & layout
**Local path:** `~/SayIt`  ¬∑  GitHub `GitCoderAccount/SayIt` (remote `origin`, branch `main`).

No build pipeline ‚Äî static files served as-is. One `SayIt` class lives in `app.js`; each feature subsystem is a per-file **augmenter** that copies its methods onto `SayIt.prototype`. **Load order** (index.html `<script>` tags): sayit-crypto ‚Üí core ‚Üí cache ‚Üí app ‚Üí settings ‚Üí profile ‚Üí polls ‚Üí notes ‚Üí spaces ‚Üí explore ‚Üí lists ‚Üí notifications ‚Üí channels ‚Üí threads ‚Üí bookmarks ‚Üí banner ‚Üí embeds ‚Üí dm.

- `index.html` ‚Äî HTML + CSS (the UI shell)
- `core.js` ‚Äî constants, `utils`, `SpaceRTC`, `DMCrypto`, URL/embed parsers, the `CHAINS` registry
- `cache.js` ‚Äî IndexedDB `Cache` layer
- `app.js` ‚Äî the `SayIt` class + bootstrap; **`SW_CACHE_VER` lives here**; the feed/render/state/virtualization core + eager wiring + core posting path
- `settings.js ¬∑ profile.js ¬∑ polls.js ¬∑ notes.js ¬∑ spaces.js ¬∑ explore.js ¬∑ lists.js ¬∑ notifications.js ¬∑ channels.js ¬∑ threads.js ¬∑ bookmarks.js ¬∑ banner.js ¬∑ embeds.js ¬∑ dm.js` ‚Äî feature augmenters (one subsystem each)
- `boot.js` ‚Äî pre-paint theme (in `<head>`)  ¬∑  `sayit-crypto.js` ‚Äî vendored SRI-pinned crypto  ¬∑  `sw.js` ‚Äî PWA service worker
- `README.md` ¬∑ `CRYPTO_BUILD.md` ¬∑ `AUDIT.md` (this file)
- `.ci/` ‚Äî extractor + ESLint config + `node --test` suite (incl. `augment.test.js`, `render.test.js`) + `smoke.py`  ¬∑  `.github/workflows/` ‚Äî `lint.yml` (lint+smoke) + `nightly.yml` (live-site smoke)

- **Boot-order race ‚Äî THE root cause of the reproducible reload "Startup failed" (2026-06-21).** Symptom: cold first load OK, then a browser reload ‚Üí feed won't paint + wallet shows disconnected. Proven live with a single clean cache (`sayitdefi-20260621-254` only ‚Äî no version mixing) yet init still threw `this._noteHTML is not a function` / `this._computeTrends is not a function` during `loadCached ‚Üí renderFeed ‚Üí init` (app.js:182), while `typeof pulse._noteHTML === 'function'` *after* load settled. Root cause: `pulse.init()` was called at app.js eval time (app.js:8831), but app.js (`<script>` #3809) loads BEFORE the augmenters that define those methods ‚Äî notes.js (#3817 ‚Üí `_noteHTML`), explore.js (#3821 ‚Üí `_computeTrends`). init paints cached posts inside its async body; on a warm reload its first IndexedDB `await` (getMuted) resolved and rendered before those later scripts ran ‚Üí augmenter method undefined ‚Üí init aborts before the background `tryAutoReconnect()` (hence the "disconnected" wallet). Only bites a returning user (cached posts in IDB) on a warm/SW-served reload ‚Äî which is why cold loads worked and headless smoke (empty IDB ‚Üí loadCached renders nothing; init error is a *caught* console.error, not a pageerror; `pulse` exists regardless) stayed green. **Fix:** defer boot ‚Äî wrap the init call in `_bootPulse()` fired on `DOMContentLoaded` (with a `document.readyState` fast-path), so every classic augmenter `<script>` has copied its methods onto the prototype before init renders. Regression guard: `.ci/test/boot-order.test.js` (3 tests ‚Äî fails if a top-level synchronous `pulse.init(` returns, or the DOMContentLoaded/readyState gate is removed). SW_CACHE_VER ‚Üí `20260621-255`. 61 unit tests + lint + smoke green. **DEPLOYED `commit d962218`; director real-browser-verified live across 3 warm reloads (feed painted 224‚Üí274‚Üí324 posts, zero init errors) and OWNER-CONFIRMED (reload works + wallet stays connected).** Lesson: a method called anywhere in init's render path is a BOOT method and must be on the prototype before init runs ‚Äî either keep it in app.js or defer init until augmenters load (the boot-order rule).
